# Controller API Conformance Discrepancies

No intentional divergences are currently accepted.

Accepted divergences must be recorded as `DISC-NNN` entries with:

- reference behavior
- Model Switchboard behavior
- impact
- resolution: `ACCEPTED`, `INVESTIGATING`, or `WILL-FIX`
- affected conformance case IDs
- review date

Accepted divergences should be represented as expected failures in the harness, not skipped cases.
